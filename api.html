md_toc.remove_md_header_syntax(header_text_line)¶

Return a trimmed version of the input line without the markdown header syntax.

Parameters:header_text_line (str) – a single markdown line that needs to be transformed into a TOC line.
Returns:a trimmed version of the input line without the markdown header syntax.
Return type:str
Raises:one of the built-in exceptions.
Note:this function removes the leading and trailing whitespaces, the first consecutive ‘#’ characters and any space left behind after removing those.
Example: 
>>> md_toc.remove_md_header_syntax(' ### hello ## how # are you ???  ! ')
'hello ## how # are you ???  !'
md_toc.get_md_header_type(line, max_header_levels=3)¶

Given a line extract the title type.

Parameters:
  • line (str) – the line to be examined.
  • max_header_levels (int) – the maximum level of headers to be considered as such when building the table of contents. Defaults to 3.
Returns:

None if the line does not contain header elements, or the header type, otherwise.
Return type:

int
Raises:

one of the built in exceptions.
Warning:

the parameter max_header_levels must be greater than 0.
Example: 
>>> print(md_toc.get_md_header_type('###  hello ## how # are you ???  ! ',6))
3
>>> print(md_toc.get_md_header_type('###  hello ## how # are you ???  ! ',2))
None
md_toc.get_md_header(header_text_line, header_duplicate_counter, max_header_levels=3, anchor_type='standard')¶

Build a data structure with the elements needed to create a TOC line.

Parameters:
  • header_text_line (str) – a single markdown line that needs to be transformed into a TOC line.
  • header_duplicate_counter (dict) – a data structure that contains the number of occurrencies of each header anchor link. This is used to avoid duplicate anchor links and it is meaningful only for certain values of anchor_type.
  • max_header_levels (int) – the maximum level of headers to be considered as such when building the table of contents. Defaults to 3.
  • anchor_type (str) – decides rules on how to generate anchor links. Defaults to standard.
Returns:

None if the input line does not correspond to one of the designated cases or a data structure containing the necessary components to create a table of contents line, otherwise.
Return type:

dict
Raises:

one of the built-in exceptions.
Note:

this works like a wrapper to other functions.
Example: 
>>> print(md_toc.get_md_header(' ## hi hOw Are YOu!!? ? #'))
{'type': 2, 'text_original': 'hi hOw Are YOu!!? ? #', 'text_anchor_link': 'hi hOw Are YOu!!? ? #'}
md_toc.build_toc_line(header, ordered=False, no_links=False, index=1)¶

Return a list element of the table of contents.

Parameters:
  • header (dict) – a data structure that contains the original text, the slugified text and the type of header.
  • ordered (bool) – if set to True, numbers will be used as list ids or otherwise a dash character, otherwise. Defaults to False.
  • no_links (bool) – disables the use of links.
  • index (int) – a number that will be used as list id in case of an ordered table of contents. Defaults to 1.
Returns:

a single line of the table of contents.
Return type:

str
Raises:

one of the built-in exceptions.
Example: 
>>> header =  {'type': 2, 'text_original': 'hi hOw Are YOu!!? ? #', 'text_anchor_link': 'hi-how-are-you'}
>>> print(md_toc.build_toc_line(header,ordered=True,index=3))
    3. [hi hOw Are YOu!!? ? #](#hi-how-are-you)

md_toc.build_anchor_link(header_text_trimmed, header_duplicate_counter, anchor_type='standard')¶

Apply the specified slug rule to build the anchor link.

Parameters:
  • header_text_trimmed (str) – the text that needs to be transformed in a link
  • header_duplicate_counter (dict) – a data structure that keeps track of possible duplicate header links in order to avoid them. This is meaningful only for certain values of anchor_type.
  • anchor_type (str) – decides rules on how to generate anchor links. Defaults to standard. Supported anchor types are: standard, github, gitlab, redcarpet.
Returns:

None if the specified anchor type is not recognized, or the anchor link, otherwise.
Return type:

str
Raises:

one of the built-in exceptions.
Note:

For a detailed description of the behaviour of each anchor type and for the licenses of each markdown parser algorithm, please refer to the ‘Markdown spec’ documentation page.
Example: 
>>> print(md_toc.build_anchor_link('   hello ## how # are you ???  ! ',dict(),'gitlab'))
hello-how-are-you
md_toc.build_toc(filename, ordered=False, no_links=False, max_header_levels=3, anchor_type='standard')¶

Parse file by line and build the table of contents.

Parameters:
  • filename (str) – the file that needs to be read.
  • ordered (bool) – decides whether to build an ordered list or not.
  • no_links (bool) – disables the use of links.
  • max_header_levels (int) – the maximum level of headers to be considered as such when building the table of contents. Defaults to 3.
  • anchor_type (str) – decides rules on how to generate anchor links. Defaults to standard.
Returns:

the corresponding table of contents.
Return type:

str
Raises:

one of the built-in exceptions.
Example: 
>>> import md_toc
>>> f = open('foo.md')
>>> print(f.read(),end='')
## Hi

hey

### How are you?           !!!

fine, thanks

# Bye
>>> print(md_toc.build_toc('foo.md',anchor_type='gitlab'),end='')
    - [Hi](#hi)
        - [How are you?           !!!](#how-are-you)
- [Bye](#bye)
