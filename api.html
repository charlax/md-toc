md_toc.get_atx_heading(line, keep_header_levels=3, parser='github', no_links=False)¶

Given a line extract the link label and its type.

Parameters:
  • line (str) – the line to be examined.
  • keep_header_levels (int) – the maximum level of headers to be considered as such when building the table of contents. Defaults to 3.
  • parser (str) – decides rules on how to generate the anchor text. Defaults to github. Supported anchor types are: github, gitlab, redcarpet.
  • no_links – disables the use of links.
Returns:

None if the line does not contain header elements according to the rules of the selected markdown parser, or a tuple containing the header type and the trimmed header text, according to the selected parser rules, otherwise.
Return type:

tuple
Raises:

one of the built in exceptions or GithubEmptyLinkLabel or GithubOverflowCharsLinkLabel
Warning:

the parameter keep_header_levels must be greater than 0.
md_toc.get_md_header(header_text_line, header_duplicate_counter, keep_header_levels=3, parser='github', no_links=False)¶

Build a data structure with the elements needed to create a TOC line.

Parameters:
  • header_text_line (str) – a single markdown line that needs to be transformed into a TOC line.
  • header_duplicate_counter (dict) – a data structure that contains the number of occurrencies of each header anchor link. This is used to avoid duplicate anchor links and it is meaningful only for certain values of parser.
  • keep_header_levels (int) – the maximum level of headers to be considered as such when building the table of contents. Defaults to 3.
  • parser (str) – decides rules on how to generate anchor links. Defaults to github.
Returns:

None if the input line does not correspond to one of the designated cases or a data structure containing the necessary components to create a table of contents line, otherwise.
Return type:

dict
Raises:

one of the built-in exceptions.
Note:

this works like a wrapper to other functions.
md_toc.build_toc_line(header, ordered=False, no_links=False, index=1, parser='github', list_marker='-')¶

Return a list element of the table of contents.

Parameters:
  • header (dict) – a data structure that contains the original text, the trimmed text and the type of header.
  • ordered (bool) – if set to True, numbers will be used as list ids or otherwise a dash character, otherwise. Defaults to False.
  • no_links (bool) – disables the use of links.
  • index (int) – a number that will be used as list id in case of an ordered table of contents. Defaults to 1.
Returns:

a single line of the table of contents.
Return type:

str
Raises:

one of the built-in exceptions.

md_toc.build_anchor_link(header_text_trimmed, header_duplicate_counter, parser='github')¶

Apply the specified slug rule to build the anchor link.

Parameters:
  • header_text_trimmed (str) – the text that needs to be transformed in a link
  • header_duplicate_counter (dict) – a data structure that keeps track of possible duplicate header links in order to avoid them. This is meaningful only for certain values of parser.
  • parser (str) – decides rules on how to generate anchor links. Defaults to github. Supported anchor types are: github, gitlab, redcarpet.
Returns:

None if the specified parser is not recognized, or the anchor link, otherwise.
Return type:

str
Raises:

one of the built-in exceptions.
Note:

The licenses of each markdown parser algorithm are reported on the ‘Markdown spec’ documentation page.
