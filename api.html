Developer Interface¶

Main Interface¶

md_toc.get_md_heading(line)¶

Given a line extract the title type and its text.

Parameter:line
Returns:None if the input line does not correspond to one of the designated cases or a data structure containing the necessary components to create a table of contents line, otherwise.
Return type:dict
Raises:one of the built-in exceptions.
Example: 
>>> print(md_toc.api.get_md_heading(' ## hi hOw Are YOu!!? ? #'))
{'type': 2, 'text_original': 'hi hOw Are YOu!!? ? #', 'text_slugified': 'hi-how-are-you'}
md_toc.build_toc_line(header, ordered=False, index=1)¶

Return a list element of the table of contents.

Parameters:
  • header (dict) – a data structure that contains the original text, the slugified text and the type of header.
  • ordered (bool) – if set to True, numbers will be used as list ids or otherwise a dash character, otherwise. Defaults to False.
  • index (int) – a number that will be used as list id in case of an ordered table of contents. Defaults to 1.
Returns:

a single line of the table of contents.
Return type:

str
Raises:

one of the built-in exceptions.
Example: 
>>> header =  {'type': 2, 'text_original': 'hi hOw Are YOu!!? ? #', 'text_slugified': 'hi-how-are-you'}
>>> print(md_toc.api.build_toc_line(header,ordered=True,index=3))
    3. [hi hOw Are YOu!!? ? #](hi-how-are-you)
md_toc.increment_index_ordered_list(header_type_count, header_type_prev, header_type_curr)¶

Compute current index for ordered list table of contents.

Parameter:header_type_count: the index numbers for all headers.
Parameter:header_type_prev: the previous type of header (h1, h2 or h3).
Parameter:header_type_curr: the current type of header (h1, h2 or h3).
Returns:None
Return type:None
Raises:one of the built in exceptions
Example: 
>>> ht = {'1':0, '2':0, '3':0}
>>> md_toc.api.increment_index_ordered_list(ht,3,3)
>>> ht
{'1': 0, '2': 0, '3': 1}
md_toc.build_toc(filename, ordered=False)¶

Parse file by line and build the table of contents.

Parameters:
  • filename (str) – the file that needs to be read.
  • ordered (bool) – decides whether to build an ordered list or not.
Returns:

the table of contents.
Return type:

str
Raises:

one of the built-in exceptions.
Example: 
>>> import md_toc
>>> f = open('foo.md')
>>> print(f.read(),end='')
# The main title

## Hello there!

[](TOC)

Here (up) will lie the toc

# Some other content ;,.-_ # bye

a b c

### zzz

# Again

## qwerty

## uiop

vbnm

### asdf

zxc

>>> print(md_toc.api.build_toc('foo.md',ordered=True),end='')
1. [The main title](the-main-title)
    1. [Hello there!](hello-there)
2. [Some other content ;,.-_ # bye](some-other-content-bye)
        1. [zzz](zzz)
3. [Again](again)
    2. [qwerty](qwerty)
    3. [uiop](uiop)
        2. [asdf](asdf)
md_toc.write_toc_on_md_file(input_file, toc, in_place=True, toc_marker='[](TOC)')¶

Write the table of contents.

Parameters:
  • input_file (str) – the file that needs to be read or modified.
  • toc (str) – the table of contents.
  • in_place (bool) – decides whether to overwrite the input file or return the table of contents. Defaults to True.
  • toc_marker (str) – a marker that will identify the start and the end of the table of contents. Defaults to [](TOC).
Returns:

None if in_place is True or the table of contents with the markers in between, otherwise.
Return type:

str
Raises:

one of the fpyutils exceptions or one of the built-in exceptions.
